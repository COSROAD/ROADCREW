## 🔴 0. 시작하기 전 — 계획부터 세우고 물어본다

프리존 님 말씀: **"물어보고 하든지요"** · **"왜 혼자 판단을 하고 하죠?"**
· **"누차 말하지만 혼자 결정하지말고 물어보라고 했습니다. 꼭 지켜주세요"**

- 무엇을 **뺄지·넣을지·어떤 방식으로** 할지 **혼자 정하지 않는다.**
- 애매하면 코드를 먼저 읽고, **계획을 세워 여쭌 뒤** 손을 댄다.
- 「글자를 뺄까요」처럼 **되돌리기 힘든 결정**은 반드시 먼저 확인.

> 오늘(로고 작업) 실패: 물어보지 않고 cosROADJOB 글자를 뺐다. 위반.

## 🔴 1. 심사숙고 — 한 번에 제대로. 대충 하고 다시 하지 않는다

프리존 님 말씀: **"여러측면에서 살펴보고 이성적으로 생각해서 심사숙고해서 하라고요"**
· **"대충하고 다시하고 하지말고요"** · **"제 발 실수하지 마세요. 시간 낭비가 심하면 안되니까요"**

- 손대기 전에 **여러 측면**을 먼저 따진다: 크기·높이·겹침·이물질·연결점·업종.
- 넣어보고→넘치면 줄이고→틀리면 다시 — **이 반복이 시간 낭비다.**
- 계산·측정으로 **미리 확인**하고 한 번에.

> 오늘 실패: 로고를 넣어보고 넘치면 줄이고, 편집띠를 반대로 잘랐다 다시. 왕복 5번.

## 🔴 2. 2차 검증 — 실수가 없어야 한다

프리존 님 말씀: **"2차 검증까지 확실히 해서 실수가 없어야 합니다"**
· **"그래서 아까 확인 하라고 한겁니다. 이런일이 생길까봐서."**

- **push 전**: `</html>` 끝 · script 짝 · node --check(classic/module) · 셀렉터 중복(@media 밖) ·
  없는 CSS 변수 · 끊어진 호출 · 함수/전역 중복 정의 · 버전==version.json
- **push 후**: **fresh git clone** 후 전체 재검사.
- **동작 검증**: 문법만으로 부족하면 **가짜 DOM/Firebase 로 실제 실행**해 화면을 그려본다.
  ⚠️ 변수 값만 보지 말 것 — **화면이 몇 번 그려지는지**를 봐야 깜빡임·덮어쓰기가 잡힌다.
  (로그인 버그·약관창 버그 둘 다 이 함정에 빠졌었다)

## 🔴 3. 연결점 체크 — 한 곳을 고치면 이어진 곳을 다시 본다

프리존 님 말씀: **"작업하면서 글자를 잘못 보거나 다른 행을 지우거나 할때는 연결점을 다시 체크하고요"**

- 클래스 이름을 새로 만들 땐 **같은 이름이 이미 있는지** 먼저 검색. (`.brand` 충돌 교훈)
- 함수·변수를 고치면 **부르는 곳 전부**를 확인. (감시자 둘·죽은 C_ICO 교훈)
- 파일을 지우면 **그 주소로 올 사람**이 있는지. (order.html 404 교훈)
- 한 앱을 고치면 **다른 앱(기사↔업체)도 같은 문제인지.** (약관창 버그 = 양쪽 다)

## 🔴 4. 업종별 문구 확인

프리존 님 말씀: **"수정 작업할때는 업종별 문구 체크해서 확인 시켜주세요"**

- COSROAD 4업종: **학원 · 물류 · 공공 · 택배**. 문구가 업종마다 다르다.
- 문구를 건드리기 전에 **업종별로 어떻게 되어 있는지 짚어 확인**받는다.
- 예: 하차 알림 호칭(학원=자녀분 / 공공=회원님), 경로 규칙(업종별 출발·도착).
